**RDBMS и NoSQL
Лабораторная работа №1**
Для СУБД PostgreSQL написать консольное приложение на Java, которое реализует следующий функционал:
DAO компоненты, поддерживающие CRUD-операции для 3 связанных сущностей;
Примеры запросов, демонстрирующих все 5 видов JOIN;
Пример запроса с использованием секций WHERE, GROUP BY, HAVING, ORDER BY и LIMIT;
Пример с использованием оконных функций;
Пример выполнения рекурсивного запроса.
Предметная область и способ работы с базой данных выбирается студентом самостоятельно (JDBC, JPA и т.д.), но структура таблиц должна позволять наглядно продемонстрировать работу всех 5 пунктов задания.

**Лабораторная работа №2**
Для СУБД ClickHouse написать консольное приложение на Java, которое реализует следующий функционал:
DAO компоненты, поддерживающие CRUD-операции для 3 связанных сущностей (таблицу с MergeTree Engine);
Примеры запросов, демонстрирующих все 5 видов JOIN;
Пример запроса с использованием секций WHERE, GROUP BY, HAVING, ORDER BY, LIMIT и LIMIT BY;
Пример работы с таблицей с ReplacingMergeTree Engine, обязательно использование столбца version для версионирования, показать работу по изменению записей через INSERT и пример запроса с получением последней версии для всех записей при наличии нескольких версий;
Пример работы с таблицей с CollapsingMergeTree Engine, обязательно использование столбца sign для механизма удаления, показать работу по удалению записей через INSERT и пример запроса с получением последней версии для всех записей при наличии нескольких версий.
Предметная область и способ работы с базой данных выбирается студентом самостоятельно (JDBC, JPA и т.д.), но структура таблиц должна позволять наглядно продемонстрировать работу всех 5 пунктов задания.

**Лабораторная работа №3**
Для СУБД Redis написать консольное приложение на Java, которое реализует следующий функционал:
DAO компоненты, поддерживающие CRUD-операции для 3 связанных сущностей, имена полей сущностей должны быть представлены в виде шаблонов “<entity_type>:<entity_id>:<field_name>”;
Запросы, демонстрирующие работу со списками в Redis (добавление и удаление элементов; пересечение и разность множеств и т.д.);
Запросы, демонстрирующие работу с множествами в Redis (аналогично все виды операций);
Запросы, демонстрирующие работу с упорядоченными множествами в Redis (аналогично все виды операций);
Сделать пункт 1 с использованием хэш-таблиц;
Продемонстрировать работу очередей (механизм publisher/subscriber).
Предметная область и способ работы с базой данных выбирается студентом самостоятельно, но они должны позволять наглядно продемонстрировать работу всех 6 пунктов задания.
